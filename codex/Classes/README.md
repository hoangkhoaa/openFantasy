## 클래스 기초

OpenFantasy의 클래스 (Classes in OpenFantasy)는 캐릭터의 타고난 능력을 활용하고 개발하는 전문화된 길을 나타냅니다.

| Concept (개념) | Description (설명) |
|---------|-------------|
| **Class Structure (클래스 구조)** | The organization of abilities and specializations (능력과 전문화의 구성) |
| **Elemental Composition (원소 구성)** | The balance of Exanthis, Soul, and Mana Capacity (엑산티스, 소울, 마나 용량의 균형) |
| **Subclasses (서브클래스)** | Specialized branches of each core class (각 핵심 클래스의 전문화된 분파) |
| **Class Progression (클래스 성장)** | The development path for class abilities (클래스 능력의 개발 경로) |
| **Equipment (장비)** | Tools and items that enhance class capabilities (클래스 능력을 향상시키는 도구 및 아이템) |

## 원소 구성

각 클래스는 능력의 형태를 결정하는 특징적인 원소 구성을 갖습니다.

| Class (클래스) | Exanthis (엑산티스) | Soul (소울) | Mana Capacity (마나 용량) | Primary Magical Affinity (주요 마법 친화력) |
|-------|----------|------|---------------|--------------------------|
| **Warrior (전사)** | 50-70% | 20-35% | 10-20% | Terramana (테라마나) (Stability (안정성)), Base Mana (기본 마나) (Force (힘)) |
| **Mage (마법사)** | 10-25% | 15-35% | 50-70% | Varies by specialization (전문화에 따라 다름) |
| **Rogue (도적)** | 30-45% | 40-60% | 10-25% | Umbramana (움브라마나) (Concealment (은폐)), Aeromana (에어로마나) (Mobility (기동성)) |
| **Cleric (성직자)** | 20-35% | 30-50% | 25-45% | Luxmana (룩스마나) (Light (빛)), Vitamana (비타마나) (Healing (치유)) |
| **Ranger (레인저)** | 35-55% | 30-45% | 15-30% | Vitamana (비타마나) (Nature (자연)), Aeromana (에어로마나) (Movement (이동)) |
| **Artificer (기공사)** | 25-40% | 20-35% | 35-55% | Crystallmana (크리스탈라마나) (Structure (구조)), Terramana (테라마나) (Solidity (견고함)) |
| **Bard (음유시인)** | 15-30% | 35-55% | 25-45% | Fulgumana (풀구마나) (Sound (소리)), Aeromana (에어로마나) (Communication (의사 소통)) |
| **Druid (드루이드)** | 20-35% | 30-45% | 30-50% | Vitamana (비타마나) (Growth (성장)), Terramana (테라마나) (Earth (땅)) |
| **Monk (수도승)** | 40-55% | 30-50% | 10-25% | Base Mana (기본 마나) (Internal Energy (내부 에너지)), Aeromana (에어로마나) (Movement (이동)) |
| **Warlock (흑마법사)** | 15-30% | 25-40% | 40-60% | Aetheramana (에테르라마나) (Void (공허)), Varies by pact (계약에 따라 다름) |

이러한 원소 균형은 특정 전문화된 마법 형태에 대한 클래스의 자연적인 친화력과 전반적인 마법 능력에 직접적인 영향을 미칩니다.

- **Specialization Efficiency (전문화 효율)**: Classes with higher Mana Capacity typically achieve better elemental specialization rates (마나 용량이 높은 클래스는 일반적으로 더 나은 원소 전문화율을 달성합니다.)
- **Mana Purification (마나 정화)**: The ratio of Soul to Mana Capacity affects a class's ability to purify Wild Mana into Base Mana (소울 대 마나 용량의 비율은 와일드 마나를 기본 마나로 정화하는 클래스의 능력에 영향을 미칩니다.)
- **Magical Stamina (마법 스테미너)**: Higher Exanthis provides better physical endurance but typically reduces magical efficiency (엑산티스가 높을수록 신체적 지구력이 향상되지만 일반적으로 마법 효율이 감소합니다.)
- **Hybrid Potential (하이브리드 잠재력)**: Classes with more balanced compositions often excel at combining different elemental types (더 균형 잡힌 구성을 가진 클래스는 종종 다양한 원소 유형을 결합하는 데 탁월합니다.)

## 문서 구조

클래스 (Classes) 디렉토리는 다음과 같이 구성됩니다.

```
codex/Classes/
├── README.md                # 이 개요 문서
├── Classes.md               # 자세한 설명이 있는 주요 클래스 문서
├── template_README.md       # 클래스 디렉토리 README 파일을 만들기 위한 템플릿
├── Warrior/                 # 예제 클래스 디렉토리
│   ├── README.md            # 클래스별 개요
│   ├── Warrior.md           # 자세한 클래스 정보
│   ├── Guardian.md          # 서브클래스 문서
│   ├── Berserker.md          # 서브클래스 문서
│   └── ...                  # 기타 서브클래스 문서
└── ...                      # 기타 클래스 디렉토리
```

### 템플릿 사용

새 클래스 (class) README를 만들려면 [template_README.md](template_README.md) 파일을 복사하고 자리 표시자 값을 클래스별 정보로 바꿉니다.

1. `[CLASS_NAME]`을 클래스 이름으로 바꿉니다.
2. 구성 범위 및 설명을 입력합니다.
3. 초점 영역 및 전문 분야와 함께 서브클래스 (subclasses)를 나열합니다.
4. 핵심 능력, 장비 및 종족 친화도를 문서화합니다.
5. 클래스 강점과 과제를 설명합니다.
